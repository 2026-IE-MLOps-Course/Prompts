Baseline rules
- load_data.py baseline:
  - Attempts to load the CSV using utils.load_csv
  - If the file does not exist, it MUST create a tiny deterministic sample CSV file at the requested path (with columns: "num_feature", "cat_feature", "target"), then load it.
  - It MUST print a LOUD, explicit message that a dummy dataset with hardcoded columns was created for scaffolding only, and that students must replace it and update their SETTINGS.
- clean_data.py baseline: returns a safe copy of the raw dataframe (identity transform), while accepting target_column
- validate.py baseline: fails fast only for obvious issues (empty df, missing required columns)
- features.py baseline:
  - Builds and returns a ColumnTransformer recipe from configuration lists.
  - Does NOT accept DataFrame data and does NOT call .fit().
  - Uses KBinsDiscretizer for quantile binning (applied only to numeric columns) and OneHotEncoder for categorical columns.
  - Must include a try/except block for OneHotEncoder (try sparse_output=False, except TypeError fallback to sparse=False) to support older scikit-learn versions.
  - Sets remainder="drop" on the ColumnTransformer.
- main.py baseline:
  - Includes a working train test split (done in main.py for clarity) BEFORE building the feature recipe to prevent leakage.
  - Uses a nested SETTINGS dictionary for configuration (acting as a bridge to YAML).
  - The SETTINGS dictionary MUST be pre-configured to work with the dummy CSV (e.g., target_column="target", numeric_passthrough=["num_feature"], categorical_onehot=["cat_feature"]) so the pipeline runs perfectly out of the box.
  - Passes the preprocessor into train_model.
- train.py baseline instantiates and fits a Pipeline:
  - Pipeline step 1 is ("preprocess", preprocessor)
  - Pipeline step 2 is ("model", estimator)
  - Ridge() for regression
  - LogisticRegression(max_iter=500) for classification
- evaluate.py baseline computes and returns:
  - RMSE (as a float) for regression
  - F1 score (as a float) for classification. For classification, use f1_score with average='weighted' so multiclass works out of the box.
  - Must call model.predict(X_test) where model is the Pipeline
- infer.py baseline calls model.predict and returns a DataFrame with a SINGLE column named "prediction" that preserves the input index (do not append it to the original DataFrame).
